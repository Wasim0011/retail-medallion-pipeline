Retail Medallion Data Pipeline: End-to-End CapstoneA production-grade batch data pipeline that ingests raw retail CSV data, applies rigorous transformations using the Medallion Architecture, and synchronizes analytical models to Snowflake.🏗️ Architecture OverviewThe project follows the Medallion Architecture to ensure data reliability and quality as it moves through the pipeline:Bronze (Raw): Ingests raw CSV files from source, persisting them as Delta Tables with added ingestion metadata.Silver (Validated): Cleans, filters, and validates data to ensure it is "Trustworthy".Gold (Analytics): Implements a Star Schema (Dimensional Modeling) optimized for business intelligence.Warehouse: Syncs final Gold models to Snowflake for downstream BI and reporting.🛠️ Tech StackProcessing: Databricks (Spark & Delta Lake) Data Warehouse: Snowflake Orchestration: Apache Airflow Languages: Python (PySpark), SQL 📁 Project Structure (Mandatory)This project adheres to a clean code folder structure:Plaintextcapstone-project/

🚀 Implementation PhasesPhase 1: Bronze (Ingestion)Reads raw CSV files and persists them as Delta Tables.Appends metadata: ingestion_timestamp and source_file_name.Maintains data in its original state with zero transformations.Phase 2: Silver (Cleaning & Validation)Customer: Removes duplicate IDs and validates email formats.Product: Ensures prices are greater than zero.Order: Filters for specific statuses and ensures non-NULL order dates.Referential Integrity: Validates that all order_items have a corresponding order_id in the orders table.Phase 3: Gold (Dimensional Modeling)Fact Sales: Joins Orders, Items, and Products; includes only DELIVERED orders.Dimensions: Flattens customer names and standardizes product catalogs.Performance: Utilizes Broadcast Joins for small dimension tables and partitioning by order_date.Phase 4: Data Quality (DQ) FrameworkAutomated checks are performed for:Uniqueness: Check for Primary Key violations in Gold tables.Completeness: Verify no NULLs in critical columns (Date, ID, Amount).Reconciliation: Record count validation between Bronze and Silver layers.⚙️ Setup & ConfigurationTo run this pipeline, you must configure the following:1. Databricks SecretsStore your credentials using Databricks Secrets to avoid hardcoding:snowflake_user, snowflake_password, snowflake_account.2. Snowflake ConnectionThe final sync uses the Spark-Snowflake Connector. Ensure the DDLs in sql/gold_tables.sql are executed in your Snowflake environment before running the pipeline.3. Airflow DAGThe pipeline is orchestrated with the following flow:
ingest_raw → build_silver → dq_check_silver → build_gold → dq_check_gold → load_snowflake.👥 ContributorsWasim - Orchestration, Snowflake Integration, & GitHub ManagementAnurag - Ingestion & Data Quality FrameworkAvranil - Dimensional Modeling & Performance Optimization
